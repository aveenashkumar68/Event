Git & GitHub

Why Git & GitHub?

Git is a powerful version control system that helps developers track changes in their code, collaborate with teammates, and roll back to earlier versions when needed. GitHub, on the other hand, is a platform that hosts Git repositories in the cloud. It allows developers to share their projects, collaborate globally, and integrate with CI/CD pipelines.

Beginner

git status ‚Äì Show changes

git add . ‚Äì Stage files

Intermediate

git checkout -b branch ‚Äì New branch

git merge branch ‚Äì Merge branch

git log --oneline ‚Äì Short log

Advanced

git reset --hard HEAD~1 ‚Äì Undo commit

git rebase main ‚Äì Reapply commits

git stash ‚Äì Save temp changes

git cherry-pick id ‚Äì Apply commit

CI & CD Pipelines

Why CI & CD?

Continuous Integration (CI) is the practice of automatically integrating code into a repository and testing it. Continuous Deployment (CD) extends this by delivering tested code into production environments.

CI (Continuous Integration)

git push origin main ‚Äì Trigger CI pipeline

mvn test ‚Äì Run automated tests (Java Example)

npm run test ‚Äì Run unit tests (Node.js Example)

CD (Continuous Deployment)

docker push repo/app ‚Äì Push to registry

kubectl apply -f deploy.yml ‚Äì Deploy to Kubernetes
